# Синхронное и асинхронное в JavaScript
# JavaScript известен своей способностью обрабатывать как синхронные, так и асинхронные операции. Понимание того, как работают эти две вещи, важно для написания эффективных, отзывчивых и удобных для пользователя приложений. В этой статье мы увидим различия между синхронным и асинхронным JavaScript с помощью наглядных примеров, которые помогут вам.

# Что такое синхронный JavaScript?
# При синхронном программировании операции выполняются одна за другой, последовательно. Таким образом, по сути, каждая строка кода ожидает завершения предыдущей, прежде чем перейти к следующей. Это означает, что программа выполняется в предсказуемом линейном порядке, при этом каждая задача завершается до начала следующей.

# В приведенном выше фрагменте кода сначала будет записана первая строка кода Hi, затем будет записана вторая строка Geek, а затем, после ее завершения, будет записана третья строка How are you. Итак, как мы видим, коды работают последовательно. Каждая строка кода ожидает, пока сначала выполнится предыдущая, а затем она выполняется.

# В синхронном коде каждый оператор ожидает завершения предыдущего перед выполнением. Это просто и легко следовать, но у него есть некоторые недостатки, особенно при выполнении трудоемких задач, таких как получение данных с сервера или чтение большого файла. Если такая задача включена в последовательность, она заблокирует выполнение остальной части кода до ее завершения, что приведет к потенциальным задержкам и ухудшению пользовательского опыта.

# Что такое асинхронный JavaScript?
# С другой стороны, асинхронное программирование позволяет выполнять несколько задач независимо друг от друга. В асинхронном коде задача может быть инициирована, и во время ожидания ее завершения могут выполняться другие задачи. Такая неблокирующая природа помогает повысить производительность и скорость реагирования, особенно в веб-приложениях.

# Итак, код сначала регистрирует Hi, затем вместо выполнения функции setTimeout он регистрирует End, а затем запускает функцию setTimeout.

# Сначала, как обычно, регистрируется оператор Hi. Поскольку мы используем браузеры для запуска JavaScript, существуют веб-API, которые обрабатывают эти вещи для пользователей. Итак, JavaScript передает функцию setTimeout в такой веб-API, а затем мы продолжаем выполнять наш код как обычно. Таким образом, он не блокирует выполнение остальной части кода, и после выполнения всего кода он помещается в стек вызовов и затем, наконец, выполняется. Вот что происходит в асинхронном JavaScript.

# Как работает асинхронный JavaScript за кулисами
# Чтобы лучше понять асинхронное поведение, важно знать о среде выполнения JavaScript, в частности о цикле событий и стеке вызовов:

![surat](/sync-vs-async-schema.png)

# Стек вызовов: Стек вызовов — это место, где функции выполняются в том порядке, в котором они вызываются. В синхронных операциях каждая функция добавляется в стек и выполняется перед переходом к следующей.
# Веб-API (в браузерах). Такие функции, как setTimeout, HTTP-запросы и прослушиватели событий, обрабатываются веб-API в браузере. Когда вызывается асинхронная функция, такая как setTimeout, она передается этим веб-API, которые управляют временем, не блокируя основной стек вызовов.
# Очередь обратного вызова: как только веб-API завершает свою работу (например, ожидает истечения тайм-аута), он помещает функцию обратного вызова (например, ту, что в setTimeout) в очередь обратного вызова.
# Цикл событий: цикл событий постоянно проверяет стек вызовов. Если он пуст, функции из очереди обратного вызова помещаются в стек для выполнения. Вот почему отложенное сообщение «Geek» регистрируется после завершения другого кода.
